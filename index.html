AI Conveyor workflow

AI Conveyor turns requests into reviewed code changes

A lightweight path from a submitted task to a GitHub Issue, Codex implementation, Pull Request, and final merge.

🚀 New visible update: Version check banner added Version: Preview test before merge

Workflow

  1. Form Submission
  2. GitHub Issue
  3. Codex CLI
  4. Pull Request
  5. Merge

How it works

F

1. Submit the form

The operator describes the change request in a small form.

I

2. Create an issue

The request becomes a structured GitHub Issue with acceptance criteria.

C

3. Let Codex implement

Codex reads the issue, edits the repository, and prepares a branch.

P

4. Review the PR

The finished branch is opened as a Pull Request for review and merge.

Current wins

  • Automatic issue creation
  • PR generation through Codex
  • Manual PR review before merge

PR Review Checklist

  • Check changed files
  • Confirm database.js was not changed
  • Confirm acceptance criteria are met
  • Choose Merge, Fix, or Reject
